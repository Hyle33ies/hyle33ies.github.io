  1. Different Objects that can be put into iter(). The order of items in a dictionary is the order in which they were added (Python 3.6+)
  2. If list appends some elements after it's itered, it's okay to call next(t). However if you change the size of dictionary, the former-created iter cannot be used again. If you just change the values, the iter is still valid.
  3. If d is a dictionary. d(=keys()), d.keys(), d.values(), d.items(return a two-element tuple) are all iterable.
  4. If you call next(), the elements in iter will be consumed. For example, the elements used will not appear in for statement, also elements being used in for statement will not appear again, They're consumed. Conversely, things like range(3,6) will not be consumed. That's to say, if r=range(3,6), you can use it over and over again.
  5. Built-in Functions for Iteration: map(func,iterable),filter(func,iterable),
